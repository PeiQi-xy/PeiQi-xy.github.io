rem布局

移动端布局

在PC端的浏览器看到的布局，放到手机浏览器中会出现无法适应大小的情况，操作很麻烦，移动端布局就是为了解决这个问题。

移动端的布局主要分为四类，分别是流式布局，rem布局，响应式布局(bootstrap)和弹性布局(flex)

rem的特点

1、可以实现页面布局中文字大小随屏幕大小变化而变化。

2、流式布局和flex布局只针对盒子宽度布局，而rem布局也可针对盒子高度布局。

3、当屏幕变化时，元素高度和宽度可以等比例缩放。

  • rem是什么

rem（font size of the root element）是指相对于根元素的字体大小的单位。简单的说它就是一个相对单位。跟当前元素的font-size没有关系，而是跟整个body的font-size有关系。

em（font size of the element）是指相对于父元素的字体大小的单位。

它们之间其实很相似，只不过rem计算的规则是依赖html元素,em是依赖父元素计算。

思考：为什么有了em还要有rem？

因为一个页面中有很多父元素，父元素中的文字大小可能不一样，通过父元素控制子盒子很难控制，但一个页面中只有一个html元素，所以我们可以将所有盒子大小都设置成rem，通过在不同屏幕下控制html元素中的文字大小来改变子盒子的大小。

  • 媒体查询

媒体查询(Media Query)是css3新语法

  • 使用media查询，可以针对不同的媒体类型定义不同的样式。
  • @media可以针对不同的屏幕尺寸设置不同的样式。
  • 当重置浏览器大小时，页面也会根据浏览器的高度和宽度重新渲染页面。
  • 目前针对很多苹果手机、Android手机、平板等设备都用得到媒体查询。

语法规范：

@media mediatype and|not|only (media feature) {

​ css—code;

}

  • mediatype 媒体类型

​ 媒体类型：将不同的终端设备划分为不同的类型。

值 解释说明
all 用于所有设备
print 用于打印机和打印预览
screen 用于电脑屏幕、平板电脑、手机等

  • 关键字

    关键字将媒体类型或多个媒体特性连接到一起作为媒体查询的条件。

    • and:可以将多个媒体特性连接到一起，相当于“且”的意思。
    • not: 排除某个媒体类型，相当于”非”的意思，可以省略。
    • only:指定某个特定的媒体类型，可以省略。

  • 媒体特性

    每种媒体类型都具备各自不同的特性，根据不同的媒体类型的媒体特性设置不同的样式。

值 解释说明
width 定义输出设备中页面可见区域的宽度
min-width 定义输出设备中页面最小可见区域的宽度（大于等于）
max-width 定义输出设备中页面最大可见区域的宽度（小于等于）

维护css的弊端

css是一门非程序式语言，没有变量、函数、SCOPE（作用域）等概念。

  • css需要书写大量看似没有逻辑的代码，冗余度比较高。
  • 不利于维护及扩展，不利于复用。
  • css没有很好的计算能力。
  • 对于非前端开发工程师来讲，往往会因为缺少 CSS 编写经验而很难写出组织良好且易于维护的 CSS 代码

  • Less

    • Less介绍

      它是一门CSS预处理（用一种专门的编程语言，为CSS增加了一些编程的特性，然后再编译成正常的CSS文

      件，以供项目使用。），扩充了CSS，增加了诸如变量、混合（mixin）、函数等功能，让 CSS 更易维护、方便制作主题、扩充，Less 也可以运行在 Node 或浏览器端，可以让我们用更少的代码做更多的事情！
      总结：Less是一门css预处理语言，它扩展了css的动态特性，让CSS更具维护性、主题性、扩展性！

      Less中文网址：http://lesscss.cn/

      常见的css预处理器：Sass、Less、Stylus

    • Less安装

      • 安装node.js，网址：https://nodejs.org/zh-cn/download/

      • 检查是否安装成功，使用cmd命令，(win10是window+r打开，运行输入cmd)，输入node -v查看版本。

      • 基于node.js安装Less，使用命令“npm install -g less”。

      • 检查是否安装成功，使用cmd命令，输入lessc -v查看版本。

    • Less使用

      首先新建一个后缀名为less的文件，在less文件里书写less语句。

    • Less变量

      变量是指可以改变的，没有固定的值。

      • @变量:值；

      • 变量命名规范：

        1、必须以@为前缀。2、不能包含特殊字符 例如: ~#等。

        3、不能以数字开头。4、大小写敏感。

    • Less编译

      浏览器不能直接读取less文件，只有将less文件编译成css文件之后html页面才可以被使用

      vscode Less 插件

      Easy LESS插件可以将less文件编译成css文件，安装完插件，重新加载下vscode，保存下less文件，会自动生成css文件。

    • Less嵌套

      css中经常用到的选择器的嵌套:

      #header .logo{

      ​ width: 300px;

      }

      Less中嵌套写法：

      #header{

      ​ .logo{

      ​ width: 300px;

      ​ }

      }

      如果遇到（交集|伪类|伪元素选择器）

      • 要有**&**符号，它才会被解析为父元素的自身或父元素的伪类。
      • 如果没有该符号，它会被解析为父选择器的后代。

    • Less 运算

      Less提供了加(+)、减(-)、乘(*)、除(/)算术运算，任何数字、颜色或者变量都可以参与运算。

      注意：1、对于两个不同的单位的值之间的运算，运算结果的值取第一个值的单位。

      2、如果两个值之间只有一个值有单位，则运算结果就取该单位。

    rem适配方案

    ​ 目标：让一些不能等比自适应的元素，达到当设备尺寸发生改变时，等比例适配当前设备。

    怎样达到目标：使用媒体查询根据不同屏幕设备按比例设置html的字体大小，然后页面元素使用rem做尺寸单 位，当html字体大小变化，元素尺寸也会发生变化，从而达到等比例缩放的适配。

    公式：html 字体大小= 屏幕宽度/划分的份数。

    ​ 页面元素的rem值= 页面元素(px)/html 字体大小。

    ​ rem实际开发适配方案1 Less + 媒体查询 + rem：

    ​ 1、动态设置html标签font-size大小

    ​ （1）、假设设计稿为750px;

    ​ （2）、假设我们把整个屏幕划分为15等份,(划分标准不固定，可以说20份也可以是10份)

    ​ （3）、这时候html字体大小就是750px/15=50px.

    ​ （4）、假设这时有一个宽高100像素的页面元素在750屏幕下，它转换成rem就是100px/html字体大小50px=2rem, 它的大小就是2rem*2rem.比例是1:1。

    ​ （5）、150屏幕下的html字体大小为150/15=15px,这时2rem=30px, 宽和高都是30px，比例还是1:1。

    ​ （6）、可以实现在不同屏幕下，页面盒子等比例缩放的效果。

    ​ rem实际开发适配开发方案2 rem + flexible.js （推荐）：

    ​ flexible.js 是手机淘宝团队出的简洁高效移动端适配库。

    ​ 有了它不需要再写不同屏幕的媒体查询，因为里面的js做了处理。

    ​ 它的原理是把当前设备划分为10等份，但是不同屏幕下，比例还是一致的。

    ​ 我们需要做的是确定好当前屏幕的html文字大小就可以了，其余的让flexible.js去算。

    ​ 下载地址：https://github.com/amfe/lib-flexible

    ​ 在html中引入flexible.js

    ​ 注意：1、flexible.js是以当前屏幕大小为准，假设设计稿宽度为750px,需要单独设置媒体查询使屏幕宽度大于750px时，html中的字体大小依然为750px/10=75px。

    ​ 2、以px为单位的页面元素转化为rem时需要下载插件cssrem

    ​ 3、cssrem中的字体默认为16px,使用该插件时需要把字头改成html中的字体大小。

    ​

演示案例网址：https://m.suning.com/
